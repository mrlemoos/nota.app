# Agent memory (nota.app)

## Workspace and stack

- Primary app is `apps/nota.app`: **Vite** client SPA (`index.html` → `app/main.tsx` → `SpaApp`), React 19, Nx monorepo; UI primitives from `@/components/ui/*` (Base UI–backed shadcn-style) and semantic theme tokens (`bg-background`, `muted`, `foreground`, etc.)—avoid hard-coded light neutrals on the notes surface.
- Notes use Supabase Postgres with RLS; **identity is Clerk** (JWT to Supabase via third-party auth); the browser uses `@supabase/supabase-js` with `accessToken` from Clerk session tokens. Data access through typed helpers in `app/models/notes.ts` rather than ad hoc queries in UI.
- TipTap edits run client-only; note body is stored as ProseMirror JSON in a `jsonb` column. The editor sync effect compares the `content` prop to the live document with ProseMirror `Node.fromJSON` + `doc.eq` (not `JSON.stringify`), since `jsonb` can reorder keys and a false diff would call `setContent` and move the caret. Also avoid pushing stale `content` into parent state after `updateNote`: the `.select()` row can lag local edits (typing during an in-flight body save, or a title save before the body debounce flushes), so merge the latest local body snapshot into the note passed to `onNoteUpdated` instead of trusting the returned row’s `content` alone.
- Public env for the app: `VITE_CLERK_PUBLISHABLE_KEY`, `VITE_SUPABASE_URL`, and `VITE_SUPABASE_ANON_KEY` (anon key only; see `apps/nota.app/.env.example`). Server-only **`CLERK_SECRET_KEY`** backs Vercel **`api/og-preview`**, Vite dev middleware for OG preview, and **`apps/nota-server`** for `verifyToken` / `authenticateRequest` and **Clerk Billing** subscription checks. **`VITE_NOTA_SERVER_API_URL`** (required for a working Nota Pro check in real environments): Nota Pro entitlement and invalidate always go to **`apps/nota-server`** with `Authorization: Bearer <Clerk session JWT>`; if the URL is missing at runtime, the client treats the user as not entitled without calling the network. Never expose `CLERK_SECRET_KEY` or Supabase service keys to the client. SQL migrations live under `supabase/migrations/` at the repo root (including Clerk third-party RLS: `0008_clerk_third_party_auth.sql`).
- Root `.gitignore` should include `supabase/.temp/`—Supabase CLI local metadata there can contain connection strings and must not be committed.
- **Stack discipline:** default stack is Nx + Vite + React (`apps/nota.app`) and Astro (`apps/nota-marketing`). Do not introduce a new framework or drive a major dependency upgrade unless the change is justified by a **user-visible** improvement or **security** fix (spell that out in the PR).

## Deployment and desktop releases

- **Vercel**: set the project **Root Directory** to `apps/nota.app`. [`apps/nota.app/vercel.json`](apps/nota.app/vercel.json) runs `npm ci` from the monorepo root and `npx nx build @nota.app/nota.app`, with **`outputDirectory`** `dist` and SPA **`rewrites`** to `index.html` (excluding `api/*` for the **`api/og-preview`** serverless handler under [`apps/nota.app/api/`](apps/nota.app/api/)). Mirror env vars from [`apps/nota.app/.env.example`](apps/nota.app/.env.example) in the Vercel dashboard (including server-only **`CLERK_SECRET_KEY`** and **`VITE_NOTA_SERVER_API_URL`** pointing at deployed **`apps/nota-server`**). Deploy **[`apps/nota-server`](apps/nota-server)** (e.g. Railway with root [`railway.json`](railway.json) + [`Dockerfile.nota-server`](Dockerfile.nota-server) and [`.dockerignore`](.dockerignore) so installs run in a clean tree, not on Railpack’s busy `node_modules` mounts) and configure **`NOTA_SERVER_CORS_ORIGINS`** on the server.
- **CI**: [`.github/workflows/ci.yml`](.github/workflows/ci.yml) runs **test + build** for `@nota.app/nota.app`; a parallel **lint** job is informational until the ESLint error backlog is cleared.
- **Electron releases**: Local and CI both use **`npx nx run @nota.app/nota-electron:electron:release`** (root script **`npm run release:electron`**), which depends on web + Electron builds then runs [`tools/electron-github-release.mjs`](tools/electron-github-release.mjs) (`electron-builder --publish always`). Set **`GH_TOKEN`** or **`GITHUB_TOKEN`** locally for GitHub Releases. [`.github/workflows/release-electron.yml`](.github/workflows/release-electron.yml) on `v*` tags or `workflow_dispatch` bumps `apps/nota-electron/package.json` then invokes that Nx target. **`electron-updater`** in [`apps/nota-electron/src/main.ts`](apps/nota-electron/src/main.ts) checks for updates when packaged; release assets must include **ZIP** and **`latest-mac.yml`**. The main process is compiled as ESM (`NodeNext`): import **`electron-updater` as default** and destructure `autoUpdater`—named ESM imports from that CJS package throw at runtime. Optional signing/notarisation via GitHub secrets (repository or **Production** environment) is documented in [`apps/nota-electron/README.md`](apps/nota-electron/README.md). The release workflow **`macos` job** uses **`environment: Production`** so **`secrets.*`** resolve **Production** environment secrets. **npm workspaces**: pin **`electronVersion`** in [`electron-builder.yml`](apps/nota-electron/electron-builder.yml) when `electron` is hoisted to the repo root; set **`npmRebuild: false`** so `electron-builder` does not run a production install in `apps/nota-electron` that prunes hoisted root `node_modules` (e.g. `app-builder-bin`). In the workflow, set **`CSC_LINK`** only when a certificate is present—an empty value breaks unsigned builds.
- **Packaged Electron app path**: packaged builds embed the **Vite client output** at `nota.app/dist` inside `app.asar` (see [`electron-builder.yml`](apps/nota-electron/electron-builder.yml)); Electron [`main.ts`](apps/nota-electron/src/main.ts) resolves it from **`app.getAppPath()`**, not `path.dirname(app.getAppPath())`. Production serves static files with a small **Node `http`** server on `127.0.0.1:4378` (SPA fallback to `index.html`). **`GET /api/og-preview`** is handled locally via a bundled **`electron-og-api.mjs`**: the Vite **`writeBundle`** step in [`vite.config.mts`](apps/nota.app/vite.config.mts) inlines **`CLERK_SECRET_KEY`** into that bundle at build time, and the main process loads **`VITE_CLERK_PUBLISHABLE_KEY`** from **`nota-public-env.json`** for cookie-based sessions—set both when running **`electron:release`**. Other **`/api/*`** paths return **502**—set **`VITE_NOTA_SERVER_API_URL`** in the **web build** that ships inside Electron so Nota Pro entitlement hits **`apps/nota-server`**. Dev still loads the Vite dev server (default port **4200**). Local preview: **`nx start @nota.app/nota.app`** runs **`vite preview`** on **4378** via [`apps/nota.app/package.json`](apps/nota.app/package.json) (no local OG API unless you add a proxy; use dev server or packaged build for OG in Electron).

## Notes UI and client state

- The signed-in **notes** experience lives in `NotesSpaShell` (`app/components/notes-spa-shell.tsx`); primary navigation uses **`app/lib/app-navigation.ts`** (location **hash**), not React Router nested routes.
- Sidebar open/closed is Zustand with `persist` to localStorage under the key `nota-notes-sidebar`.
- The `/notes` layout row fills the viewport (`h-dvh` with `min-h-0` on the shell, `aside`, and `main`); only `main` scrolls the document, and the sidebar note list scrolls inside its own `flex-1 min-h-0 overflow-y-auto` region so the window does not grow with note height. In Electron (`titleBarStyle: 'hiddenInset'` in `apps/nota-electron`), `useIsElectron()` (detects preload `window.nota` or UA; modern Electron often omits the Electron token from the UA) drives traffic-light clearance on `/notes`: sidebar header uses `pl-20` and safe-area top padding; when the sidebar is collapsed the expand control is `position: fixed` in a `min-h-[52px]` `items-center` strip (`pl-20`, safe-area `pt`) with `pointer-events-none` on the wrapper and `pointer-events-auto` on the button. `main` uses the same top padding whether the sidebar is open or closed so toggling does not shift scroll layout.
- There is no global top nav. Note Graph and Settings (`/notes/settings`) links live in the notes sidebar footer (`routes/notes.tsx`); theme, today’s-note shortcut, account email, and sign out are on the Settings page. When the inline document title scrolls out of the notes layout `main` (`overflow-auto`, not the window), a read-only centered title appears as a fixed overlay at the top of the viewport (`routes/notes.tsx`) driven by `app/context/sticky-doc-title.tsx` and an `IntersectionObserver` rooted on that `main`.
- `useStickyDocTitle` must not throw when `StickyDocTitleProvider` is missing (e.g. Vitest `createRoutesStub` without root `Layout`); use a no-op fallback so partial trees still render.
- Note list sidebar heading and the `/notes` empty-state title use serif (`font-serif` / Instrument Serif) with `tracking-normal`; the open note’s document title field stays sans.
- Note editor is a centered document column (no bordered card around TipTap); the inline title is a debounced, wrapping textarea with auto height (saved to DB `title`); empty display label matches `persistedDisplayTitle` in `app/lib/note-title.ts` (“Untitled Note”). Body autosave must update `content` only and must not overwrite `title` from the first paragraph of the editor.
- After successful client-side saves from `NoteEditor`, refresh list metadata via the notes data layer (e.g. `refreshNotesList` / context updates) so sidebar titles and dates stay in sync.
- Delete note: from the trash control on each sidebar row (with confirm) and from the Command Palette when a note is open—**not** from a control beside the document title. Implementation uses **`spaDeleteNoteById`** (`app/lib/spa-delete-note.ts`) so IndexedDB and Supabase stay aligned.
- Create note: **`spaCreateNote`** (`app/lib/spa-create-note.ts`) from the notes shell and Command Palette; keep navigation/hash updates consistent with **`app/lib/app-navigation.ts`** when inserting the new id.
- Command palette (`app/components/command-palette.tsx`) mounts from `app/signed-in-command-palette.tsx` under the signed-in notes tree. Cmd/Ctrl+K toggles it from anywhere (including the TipTap body and title field); it lists notes from **`NotesData`**, opens notes via hash navigation, creates/deletes through the SPA helpers above, and signs out via Clerk (`useClerk().signOut()` + hash navigation)—not React Router `Form` posts.
- Micro-motion uses **`gsap`** and **`@gsap/react`** via `app/lib/nota-motion.ts` (`usePrefersReducedMotion`, shared eases/durations). Command palette enter/exit is GSAP-driven with Base UI Dialog `actionsRef`, `preventUnmountOnClose()` on close, and `unmount()` after the exit timeline; notes sidebar open/close tweens the `<aside>` `maxWidth`/`opacity`; shared `Button` adds a short press scale. **`vite.config.mts`** sets `ssr.noExternal: ['gsap', '@gsap/react']` so Vite bundles them for SSR—otherwise Node would load GSAP’s ESM package from `node_modules` on Vercel and crash with “Cannot use import statement outside a module”. Avoid GSAP on the TipTap document surface (caret and performance).

## Learned User Preferences

- PDF attachments should live inline in the TipTap document (insert at the cursor as a `notePdf` block) rather than only in a separate footer panel.
- The open note’s inline document title textarea should use a heavy sans weight and `text-pretty` so large, multi-line titles read clearly.
- Link to another note from the editor body by typing `@` (filterable mention menu), not a **Link to note** toolbar control.
- Prefer a minimal attachment entry in the editor chrome: no **Insert PDF or image** button or long instructional hint in the toolbar row; rely on drag-and-drop for PDFs and images while still surfacing upload errors when needed.
- After inserting an internal note link from the `@` flow, continue in normal paragraph typing without extending the link mark (e.g. `setParagraph` and clear stored marks after insert).
- Product and public docs favour a calm, direct voice: position nota.app against compulsive attention-economy patterns in thinking tools (not a blanket anti-AI stance); no AI-generated writing in the product experience and no engagement-bait UI.
- Public-facing copy (marketing site, store listings) should match product behaviour: there is **no unpaid tier or free trial**—users download the Mac app, sign in, and **subscribe in-app** for full access, cloud sync, and backup; marketing must not promise a free full vault. Core purchase flows stay in-app after install, not as a web-only checkout on the notes surface. On **Settings**, the Nota Pro / subscription block should foreground **direct plan selection and checkout** (Clerk Billing UI) rather than long marketing copy.
- Notes should be offline-first / local-first: capture edits locally and sync to the cloud; when the app starts with network available, load notes and attachments from the server.
- Optional “today’s note” shortcut (Mod+D when enabled): no streaks, reminders, or other gamification; command palette can surface the same action with a matching hotkey hint when the setting is on. New daily notes use the local long date as the initial title (e.g. 4 March 2026), not “Untitled Note”.
- Shared `Button` is Base UI–backed and does not support Radix-style `asChild`; for controls that must behave as real links (e.g. billing portal URLs in new tabs), style an `<a>` with exported `buttonVariants` and `cn()`, or use `Button` + `window.open` if trading off middle-click and copy-link behaviour is acceptable.
- Keep the in-app and favicon mark minimal and in sync (`NotaLogo` ↔ `public/favicon.svg`); after changing geometry run **`npm run generate:nota-icons`** for `apple-touch-icon.png` and, on macOS, `apps/nota-electron/buildResources/icon.icns`.

## Learned Workspace Facts

- **`NotesDataProvider`** (`app/context/notes-data-context.tsx`) calls **`fetchNotaProEntitled`** ([`app/lib/nota-server-client.ts`](apps/nota.app/app/lib/nota-server-client.ts): **`apps/nota-server`** at **`VITE_NOTA_SERVER_API_URL`** with Bearer JWT; missing base URL → 401-style handling without a network call) and exposes **`notaProEntitled`** (meaning **licensed / subscribed**) plus `refreshNotesList` / patch helpers. When **`notaProEntitled`** is true, it loads Supabase notes and `user_preferences`, merges with IndexedDB via `app/lib/notes-offline/`. When false, the user gets **no vault**: notes list stays empty (no IndexedDB bootstrap for unpaid users). Offline continuation uses **`readNotaServerEntitledSession`** so previously entitled users can still read local cache when the network or entitlement request fails. Signed-in but unpaid users see **`NotesSpaShell`** paywall UI (sidebar hidden, hash steered to **Settings** to subscribe). **`NoteDetailPanel`** / uploads / cloud saves require entitlement. Per-note **`editor_settings`** (`jsonb`) syncs when entitled and online. When entitled after load with an empty vault (`notes.length === 0`) and `user_preferences.welcome_seeded` is false, **`runWelcomeNoteSeedIfNeeded`** (`welcome-note-seed.ts`) seeds **`welcome-note-doc`** via **`createLocalOnlyNote`**, persists **`welcome_seeded`** with **`upsertUserPreferences`**, a per-user promise map to avoid Strict Mode double-insert, then **`refreshNotesList`** and **`setAppHash`** to open the welcome note. **Clerk Billing** server truth lives in **`app/lib/clerk-billing.server.ts`** (cached; fail closed); checkout in **Settings** via **`PricingTable`** where enabled; after purchase call **`postNotaProInvalidate`** and **`refreshNotesList`**; configure plans in the **Clerk Dashboard** (Stripe under the hood; not synced from legacy RevenueCat). **`refreshNotesList({ silent: true })`** refreshes list data **without** flipping global **`loading`** / **`shellReady`**—use it for background work (offline outbox follow-ups, editor revalidate, create/delete tails, command palette) so **`useNotesOfflineSync`** (gated by **`shellReady`**) does not thrash; keep non-silent refresh for explicit user feedback (e.g. Settings “refresh subscription”); reset refresh sequencing when **`userId`** changes.
- Per-note attachments use Supabase Storage bucket `note-pdfs` (PDFs plus allowlisted raster images per migration), `note_attachments` rows, and TipTap `notePdf` / `noteImage` atom blocks in `notes.content` JSON (`attachmentId`, `filename`); generalised `noteAttachmentStoragePath` and shared upload helpers in `app/lib/pdf-attachment-client.ts`. Drag-and-drop uploads require **`notaProEntitled`** (`tiptap-editor.tsx`). RLS allows owners to update `note_attachments.filename` for display-name renames on PDF nodes (double-click the label). Image blocks resolve signed URLs and refresh before expiry; signed-URL logic should depend on stable fields such as `storage_path`, not the whole attachment row object, so loader revalidation (new object references) does not refetch and flicker the image. PDF modal preview uses PDF.js in a lazy-loaded chunk (`pdf-js-modal-preview.tsx`); keep PDF.js off the eager import path for `note-pdf-extension.tsx` (use `React.lazy` and `Suspense`) or `notePdf` nodes can vanish. Render the preview `<dialog>` with `createPortal` to `document.body`. `app/lib/pdf-preview-url.ts` adds `#toolbar=0&navpanes=0` for iframe fallback only when PDF.js fails. File insert uses drag-and-drop with ProseMirror `editorProps` drop handling on the real editor surface when no file picker row is shown. Deleting a note: `deleteNote` in `app/models/notes.ts` lists `note_attachments`, removes objects from Storage (`note-pdfs`) while the note row still exists (storage RLS), then deletes the note; attachment rows drop via `ON DELETE CASCADE`.
- Link previews use an authenticated **`GET /api/og-preview?url=`** handler (`api/og-preview.ts` / dev middleware) and `app/lib/og-preview.server.ts` for server-side fetch and Open Graph parsing; the editor exposes a TipTap `linkPreview` block and debounced conversion of link-only paragraphs via `app/components/tiptap/link-preview-scan.ts`. If OG fetch fails or returns no usable title/description/image, revert to a normal paragraph with a link; the link mark uses `skipLinkPreview` so the scanner does not immediately re-promote the same URL and loop. Paragraphs whose only link href is an internal note path `/notes/<uuid>` are not promoted to `linkPreview`.
- Internal note links: `href` is path-only `/notes/<uuid>` (see `app/lib/internal-note-link.ts`); `NotaLink` uses `openOnClick: false` with a custom click handler—internal paths update the in-app hash / navigation, modifier-click opens a new tab, external URLs use `window.open`; `@` mention menus use the live notes list from editor/`NotesData` context. Forward/back indexes for the vault are derived from TipTap JSON via `app/lib/note-link-graph.ts` (no separate links table for v1); the graph surface is lazy-loaded React Flow, and `NoteBacklinksPanel` lists inbound links on the open note. The graph view (`notes-graph-view.tsx`) uses **`filterNotesForNoteGraph`** so notes with **`editor_settings.showInNoteGraph === false`** are omitted; **Show in note graph** in the note layout menu (`note-layout-menu.tsx`) toggles that flag (backlinks still list all notes).
- Mermaid in notes uses `NotaCodeBlock` (`app/components/tiptap/nota-code-block.tsx`): `StarterKit.configure({ codeBlock: false })` plus `CodeBlockLowlight` from `@tiptap/extension-code-block-lowlight` with a shared `notaLowlight` instance (`app/lib/nota-lowlight.ts`: `createLowlight(common)` and register `mermaid` as plaintext so diagram fences are not auto-highlighted); React node view for the block. When `attrs.language` is `mermaid` (case-insensitive), lazy-load `mermaid`, debounce renders, use `securityLevel: 'strict'`, and derive light/dark theme from the `<html>` `dark` class. Non-mermaid fences get lowlight `hljs-*` decorations; style tokens in `styles.css` under `.tiptap-editor .tiptap-code-block` for light and `.dark`. Fence input rules allow broader language ids (e.g. `c++`) than stock TipTap. In `TipTapEditor`, declare hooks such as `useCallback` for toolbar actions before any early return that skips the mounted editor tree so hook order stays valid.
- TipTap body sync and editor instance stability: apply `content` from props with `setContent` only when `noteId` changes (user opened a different note), not on every parent update after save or list refresh for the same note—otherwise the selection can jump. After a successful body save, align `lastSavedContent` with the merged local body snapshot passed to `onNoteUpdated`, not only the payload that triggered the request. `useEditor` depends on `extensions`; callbacks captured in the `extensions` `useMemo` (e.g. `onSaveDueDate`) must stay referentially stable when parent `note` updates after saves—otherwise `extensions` is recreated, the editor instance is destroyed, and the caret jumps. Stabilise with refs for the latest `note` / handlers and `useCallback(..., [])` (as with `persistDueDate` in `note-editor.tsx`). For `@` note-mention confirmation (`editorProps.handleKeyDown` and `beforeinput` for Return), pass the same `EditorView` into `tryConfirmNoteMention` and insert the link with `view.dispatch` (delete the `@…` span, insert linked text, selection + `setStoredMarks([])`), not only `Editor.chain()` via a React ref that can lag the active view; treat `NumpadEnter` like `Enter` for confirm. Never `preventDefault()` on confirm keys without inserting, or Enter can be swallowed with no link.
- Signed-in command palette: when open, Cmd/Ctrl+N submits create-note **only if entitled** (if not busy); Space focuses the filter unless the input is already focused; create-note and sign-out close the palette; when **not** entitled, a **Subscription** group offers “Open Settings to subscribe” instead of note actions. Muted right-side labels can show hotkeys (e.g. ⌘N vs Ctrl+N from platform sniff). Menu rows use `HugeiconsIcon` with icons from `@hugeicons/core-free-icons` for consistent item affordances. The palette list hides scrollbars while staying scrollable (`[scrollbar-width:none]`, `[-ms-overflow-style:none]`, `[&::-webkit-scrollbar]:hidden` on `Command.List` in `command-palette.tsx`); the notes layout document `<main>` uses the same scrollbar-hiding utilities. `Dialog.Popup` carries `data-nota-command-palette` so global Mod+D for “today’s note” does not run while the palette is focused.
- TipTap **tables** and **emoji**: register `@tiptap/extension-table`, `table-row`, `table-header`, and `table-cell` in `tiptap-editor.tsx`; insert tables via the command palette (`note-editor-commands`, `insertTable`). Row/column/header/table delete actions live in `tiptap/table-editor-menu.tsx` (`BubbleMenu`); use `mousedown` capture + `preventDefault` on controls so ProseMirror `CellSelection` survives until `deleteRow` / `deleteColumn` run. Table styling is in `styles.css` (e.g. `--nota-table-grid`); header-corner CSS must target `thead` / `tbody` because TipTap’s table DOM puts `<colgroup>` before row groups. Emoji: `@tiptap/extension-emoji` on the **v2** TipTap line (`^2.27.2`); `enableEmoticons: false`; `suggestion: { allow: () => false }` disables the `:` menu while keeping shortcode/Unicode. Style `span[data-type='emoji']` under `.tiptap-editor` if needed. Vitest: stub `HTMLCanvasElement.prototype.getContext` in `vitest.setup.ts` for `is-emoji-supported`; colocate `tiptap-emoji.spec.ts` with the editor.
- Notes **SPA shell**: graph, settings, shortcuts, list, and note panels mount as siblings under `ShellPanel` in `notes-spa-shell.tsx`; inactive panels use Tailwind **`hidden`** with **`inert`** and **`aria-hidden`** so the notes data tree and editor stay mounted when switching surfaces (avoiding repeated Supabase/IndexedDB hydration from full remounts). When the user is signed in but **not** entitled, the shell **hides the sidebar**, shows a subscription banner, and **`replaceAppHash`** steers navigation to **Settings** (purchase). Unrecognised hash routes resolve to **`SpaNotFound`**; use canonical **`#/404`** for **`hashForScreen({ kind: 'notFound' })`** / **`replaceAppHash`** (not `#/`, which parses as landing and redirects signed-in users to notes). After a missing note finishes loading, **`NoteDetailPanel`** calls **`replaceAppHash({ kind: 'notFound' })`** instead of leaving “Note not found or still loading…”. First-request unknown pathnames return HTTP **404** from **`apps/nota.app/middleware.ts`** (Vercel Edge) using **`isSpaShellPathnameAllowed`** in **`spa-pathname-policy.ts`**; packaged Electron’s static server uses the same allowlist for SPA fallback vs **404**. **Clerk on a hash SPA:** configure **`ClerkProvider`** with **`signInUrl`/`signUpUrl`** and **`routerPush`/`routerReplace`** so flows stay on in-app hash auth routes instead of bouncing to the hosted Account Portal; hyphenated **`/sign-in`/`/sign-up`** substeps and **`?…`** on the hash must parse as auth, not **`notFound`**. **`history.pushState`/`replaceState`** do not fire **`hashchange`**—**`app-navigation.ts`** patches them to **`notify`** shell subscribers; use **`queueMicrotask`** before **`notify`** when history changes during another component’s render, while **`replaceAppHash`** from layout effects can stay synchronous. In **`SpaApp`**, keep hooks **above** any **`loading`** early return so hook order never changes. Prefer **`ClerkProvider` outside `React.StrictMode`** in dev to avoid remount-driven session flashes. Custom sign-in and sign-up screens use **`@clerk/elements`** primitives so the UI is not coupled to Clerk’s prebuilt `<SignIn />`/`<SignUp />` form behaviour; **Google** is the only social provider alongside **email/password** (Apple, GitHub, passkey-only, and username flows are out of scope for this product).

## General Guidelines for working with Nx

- For navigating/exploring the workspace, invoke the `nx-workspace` skill first - it has patterns for querying projects, targets, and dependencies
- When running tasks (for example build, lint, test, e2e, etc.), always prefer running the task through `nx` (i.e. `nx run`, `nx run-many`, `nx affected`) instead of using the underlying tooling directly
- Prefix nx commands with the workspace's package manager (e.g., `pnpm nx build`, `npm exec nx test`) - avoids using globally installed CLI
- You have access to the Nx MCP server and its tools, use them to help the user
- For Nx plugin best practices, check `node_modules/@nx/<plugin>/PLUGIN.md`. Not all plugins have this file - proceed without it if unavailable.
- NEVER guess CLI flags - always check nx_docs or `--help` first when unsure

## Scaffolding & Generators
