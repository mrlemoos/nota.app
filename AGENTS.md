# Agent memory (nota.app)

## Workspace and stack

- Primary app is `apps/nota.app`: React Router 7 (SSR), Vite, React 19, Nx monorepo; UI primitives from `@/components/ui/*` (Base UI–backed shadcn-style) and semantic theme tokens (`bg-background`, `muted`, `foreground`, etc.)—avoid hard-coded light neutrals on `/notes` routes.
- Notes use Supabase Postgres with RLS; session auth via `@supabase/ssr` and HTTP cookies; data access through typed helpers in `app/models/notes.ts` rather than ad hoc queries in UI.
- TipTap edits run client-only; note body is stored as ProseMirror JSON in a `jsonb` column. The editor sync effect compares the `content` prop to the live document with ProseMirror `Node.fromJSON` + `doc.eq` (not `JSON.stringify`), since `jsonb` can reorder keys and a false diff would call `setContent` and move the caret. Also avoid pushing stale `content` into parent state after `updateNote`: the `.select()` row can lag local edits (typing during an in-flight body save, or a title save before the body debounce flushes), so merge the latest local body snapshot into the note passed to `onNoteUpdated` instead of trusting the returned row’s `content` alone.
- Public env for the app: `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` (anon key only); RevenueCat Web Billing public SDK key `VITE_REVENUECAT_API_KEY` (see `apps/nota.app/.env.example`). Server-side RevenueCat REST checks use `REVENUECAT_SECRET_API_KEY` only on the server—never expose it to the client or document real key values here. SQL migrations live under `supabase/migrations/` at the repo root.
- Root `.gitignore` should include `supabase/.temp/`—Supabase CLI local metadata there can contain connection strings and must not be committed.

## Deployment and desktop releases

- **Vercel**: set the project **Root Directory** to `apps/nota.app`. [`apps/nota.app/vercel.json`](apps/nota.app/vercel.json) runs `npm ci` from the monorepo root and `npx nx build @nota.app/nota.app`. SSR uses **`@vercel/react-router`** (`vercelPreset` in [`react-router.config.ts`](apps/nota.app/react-router.config.ts)). Mirror env vars from [`apps/nota.app/.env.example`](apps/nota.app/.env.example) in the Vercel dashboard (including server-only `REVENUECAT_SECRET_API_KEY`).
- **CI**: [`.github/workflows/ci.yml`](.github/workflows/ci.yml) runs **test + build** for `@nota.app/nota.app`; a parallel **lint** job is informational until the ESLint error backlog is cleared.
- **Electron releases**: [`.github/workflows/release-electron.yml`](.github/workflows/release-electron.yml) on `v*` tags or `workflow_dispatch` builds macOS artefacts and publishes to **GitHub Releases** (`electron-builder --publish always`). **`electron-updater`** in [`apps/nota-electron/src/main.ts`](apps/nota-electron/src/main.ts) checks for updates when packaged; release assets must include **ZIP** and **`latest-mac.yml`**. Optional signing/notarisation via repository secrets is documented in [`apps/nota-electron/README.md`](apps/nota-electron/README.md).
- **Packaged Electron server path**: production `react-router build` may emit `build/server/<runtime>/index.js` (e.g. with the Vercel preset); Electron [`main.ts`](apps/nota-electron/src/main.ts) resolves nested `index.js` under `build/server`. Local **`nx start @nota.app/nota.app`** uses the same discovery via an `nx` target override in [`apps/nota.app/package.json`](apps/nota.app/package.json).

## Notes UI and client state

- `/notes` uses a parent layout route with nested index and `:noteId` child routes.
- Sidebar open/closed is Zustand with `persist` to localStorage under the key `nota-notes-sidebar`.
- The `/notes` layout row fills the viewport (`h-dvh` with `min-h-0` on the shell, `aside`, and `main`); only `main` scrolls the document, and the sidebar note list scrolls inside its own `flex-1 min-h-0 overflow-y-auto` region so the window does not grow with note height. In Electron (`titleBarStyle: 'hiddenInset'` in `apps/nota-electron`), `useIsElectron()` (detects preload `window.nota` or UA; modern Electron often omits the Electron token from the UA) drives traffic-light clearance on `/notes`: sidebar header uses `pl-20` and safe-area top padding; when the sidebar is collapsed the expand control is `position: fixed` in a `min-h-[52px]` `items-center` strip (`pl-20`, safe-area `pt`) with `pointer-events-none` on the wrapper and `pointer-events-auto` on the button. `main` uses the same top padding whether the sidebar is open or closed so toggling does not shift scroll layout.
- There is no global top nav. Note Graph and Settings (`/notes/settings`) links live in the notes sidebar footer (`routes/notes.tsx`); theme, today’s-note shortcut, account email, and sign out are on the Settings page. When the inline document title scrolls out of the notes layout `main` (`overflow-auto`, not the window), a read-only centered title appears as a fixed overlay at the top of the viewport (`routes/notes.tsx`) driven by `app/context/sticky-doc-title.tsx` and an `IntersectionObserver` rooted on that `main`.
- `useStickyDocTitle` must not throw when `StickyDocTitleProvider` is missing (e.g. Vitest `createRoutesStub` without root `Layout`); use a no-op fallback so partial trees still render.
- Note list sidebar heading and the `/notes` empty-state title use serif (`font-serif` / Instrument Serif) with `tracking-normal`; the open note’s document title field stays sans.
- Note editor is a centered document column (no bordered card around TipTap); the inline title is a debounced, wrapping textarea with auto height (saved to DB `title`); empty display label matches `persistedDisplayTitle` in `app/lib/note-title.ts` (“Untitled Note”). Body autosave must update `content` only and must not overwrite `title` from the first paragraph of the editor.
- After successful client-side saves from `NoteEditor`, the note detail route should call React Router `revalidate()` (e.g. `useRevalidator`) so the parent `notes` layout loader refreshes sidebar titles and dates.
- Delete note: POST `/notes/:noteId` from the trash control on each sidebar row (with confirm) and from the Command Palette when that route is active—not from a control beside the document title.
- Create-note `<Form>` controls in the notes layout (`routes/notes.tsx`) must set `action="/notes"` so POST always hits the layout `action`; React Router’s `Form` defaults to the current pathname, which wrongly targets `/notes/graph` (no create action) or `/notes/:noteId` (delete) when those routes are active.
- Command palette (`app/components/command-palette.tsx`) mounts from `app/signed-in-command-palette.tsx` in the root layout when a user session exists. Cmd/Ctrl+K toggles it from anywhere (including the TipTap body and title field); it lists notes from the `/notes` layout loader to open one (navigate to `/notes/:noteId`), POSTs to `/notes` to create a note, to `/notes/:noteId` to delete the open note, and to `/logout` for sign out.
- Micro-motion uses **`gsap`** and **`@gsap/react`** via `app/lib/nota-motion.ts` (`usePrefersReducedMotion`, shared eases/durations). Command palette enter/exit is GSAP-driven with Base UI Dialog `actionsRef`, `preventUnmountOnClose()` on close, and `unmount()` after the exit timeline; notes sidebar open/close tweens the `<aside>` `maxWidth`/`opacity`; shared `Button` adds a short press scale. Avoid GSAP on the TipTap document surface (caret and performance).

## Learned User Preferences

- PDF attachments should live inline in the TipTap document (insert at the cursor as a `notePdf` block) rather than only in a separate footer panel.
- The open note’s inline document title textarea should use a heavy sans weight and `text-pretty` so large, multi-line titles read clearly.
- Link to another note from the editor body by typing `@` (filterable mention menu), not a **Link to note** toolbar control.
- Prefer a minimal attachment entry in the editor chrome: no **Insert PDF or image** button or long instructional hint in the toolbar row; rely on drag-and-drop for PDFs and images while still surfacing upload errors when needed.
- After inserting an internal note link from the `@` flow, continue in normal paragraph typing without extending the link mark (e.g. `setParagraph` and clear stored marks after insert).
- Product and public docs favour a calm, direct voice: position nota.app against compulsive attention-economy patterns in thinking tools (not a blanket anti-AI stance); no AI-generated writing in the product experience and no engagement-bait UI.
- Notes should be offline-first / local-first: capture edits locally and sync to the cloud; when the app starts with network available, load notes and attachments from the server.
- Optional “today’s note” shortcut (Mod+D when enabled): no streaks, reminders, or other gamification; command palette can surface the same action with a matching hotkey hint when the setting is on.

## Learned Workspace Facts

- The `/notes` client loader merges server-fetched notes with local/offline state via `app/lib/notes-offline/` (IndexedDB, outbox, `merge-note-with-local.ts` and related modules) and the client branch of `routes/notes.tsx`, so edits can persist locally and reconcile with Supabase when online. If create fails on the server and the client redirects to a local-only note (`pending_create`), the note detail client loader in `routes/notes.$noteId.tsx` should still return that row from IndexedDB before treating a missing server row as 404 while the browser reports online. Per-note `editor_settings` (`jsonb` on `notes`: font sans/serif/mono, measure narrow/default/wide) follows the same local draft, merge, and sync path as other scalar fields.
- Per-note attachments use Supabase Storage bucket `note-pdfs` (PDFs plus allowlisted raster images per migration), `note_attachments` rows, and TipTap `notePdf` / `noteImage` atom blocks in `notes.content` JSON (`attachmentId`, `filename`); generalised `noteAttachmentStoragePath` and shared upload helpers in `app/lib/pdf-attachment-client.ts`. RLS allows owners to update `note_attachments.filename` for display-name renames on PDF nodes (double-click the label). Image blocks resolve signed URLs and refresh before expiry; signed-URL logic should depend on stable fields such as `storage_path`, not the whole attachment row object, so loader revalidation (new object references) does not refetch and flicker the image. PDF modal preview uses PDF.js in a lazy-loaded chunk (`pdf-js-modal-preview.tsx`); keep PDF.js off the eager import path for `note-pdf-extension.tsx` (use `React.lazy` and `Suspense`) or `notePdf` nodes can vanish. Render the preview `<dialog>` with `createPortal` to `document.body`. `app/lib/pdf-preview-url.ts` adds `#toolbar=0&navpanes=0` for iframe fallback only when PDF.js fails. File insert uses drag-and-drop with ProseMirror `editorProps` drop handling on the real editor surface when no file picker row is shown. Deleting a note: `deleteNote` in `app/models/notes.ts` lists `note_attachments`, removes objects from Storage (`note-pdfs`) while the note row still exists (storage RLS), then deletes the note; attachment rows drop via `ON DELETE CASCADE`.
- Link previews use an authenticated `GET /og-preview?url=` resource route and `app/lib/og-preview.server.ts` for server-side fetch and Open Graph parsing; the editor exposes a TipTap `linkPreview` block and debounced conversion of link-only paragraphs via `app/components/tiptap/link-preview-scan.ts`. If OG fetch fails or returns no usable title/description/image, revert to a normal paragraph with a link; the link mark uses `skipLinkPreview` so the scanner does not immediately re-promote the same URL and loop. Paragraphs whose only link href is an internal note path `/notes/<uuid>` are not promoted to `linkPreview`.
- Internal note links: `href` is path-only `/notes/<uuid>` (see `app/lib/internal-note-link.ts`); `NotaLink` uses `openOnClick: false` with a custom click handler—internal paths use React Router `navigate`, modifier-click opens a new tab, external URLs use `window.open`; notes for pickers come from `notesFromMatches` in `app/lib/notes-from-matches.ts`. Forward/back indexes for the vault are derived from TipTap JSON via `app/lib/note-link-graph.ts` (no separate links table for v1); `/notes/graph` shows a lazy-loaded React Flow view, and `NoteBacklinksPanel` lists inbound links on the open note.
- Mermaid in notes uses `NotaCodeBlock` (`app/components/tiptap/nota-code-block.tsx`): `StarterKit.configure({ codeBlock: false })` plus `CodeBlockLowlight` from `@tiptap/extension-code-block-lowlight` with a shared `notaLowlight` instance (`app/lib/nota-lowlight.ts`: `createLowlight(common)` and register `mermaid` as plaintext so diagram fences are not auto-highlighted); React node view for the block. When `attrs.language` is `mermaid` (case-insensitive), lazy-load `mermaid`, debounce renders, use `securityLevel: 'strict'`, and derive light/dark theme from the `<html>` `dark` class. Non-mermaid fences get lowlight `hljs-*` decorations; style tokens in `styles.css` under `.tiptap-editor .tiptap-code-block` for light and `.dark`. Fence input rules allow broader language ids (e.g. `c++`) than stock TipTap. In `TipTapEditor`, declare hooks such as `useCallback` for toolbar actions before any early return that skips the mounted editor tree so hook order stays valid.
- TipTap body sync and editor instance stability: apply `content` from props with `setContent` only when `noteId` changes (user opened a different note), not on every parent update after save or `revalidate` for the same note—otherwise the selection can jump. After a successful body save, align `lastSavedContent` with the merged local body snapshot passed to `onNoteUpdated`, not only the payload that triggered the request. `useEditor` depends on `extensions`; callbacks captured in the `extensions` `useMemo` (e.g. `onSaveDueDate`) must stay referentially stable when parent `note` updates after saves—otherwise `extensions` is recreated, the editor instance is destroyed, and the caret jumps. Stabilise with refs for the latest `note` / handlers and `useCallback(..., [])` (as with `persistDueDate` in `note-editor.tsx`).
- Signed-in command palette: when open, Cmd/Ctrl+N submits create-note (if not busy) and Space focuses the filter unless the input is already focused; create-note and sign-out close the palette; muted right-side labels can show hotkeys (e.g. ⌘N vs Ctrl+N from platform sniff). Menu rows use `HugeiconsIcon` with icons from `@hugeicons/core-free-icons` for consistent item affordances. The palette list hides scrollbars while staying scrollable (`[scrollbar-width:none]`, `[-ms-overflow-style:none]`, `[&::-webkit-scrollbar]:hidden` on `Command.List` in `command-palette.tsx`); the notes layout document `<main>` uses the same scrollbar-hiding utilities. `Dialog.Popup` carries `data-nota-command-palette` so global Mod+D for “today’s note” does not run while the palette is focused.
- TipTap tables: register `@tiptap/extension-table`, `table-row`, `table-header`, and `table-cell` in `tiptap-editor.tsx` (versions aligned with the rest of TipTap); insert via the command palette through `note-editor-commands` (`insertTable` registration). Row/column/header/table delete actions live in `tiptap/table-editor-menu.tsx` (`BubbleMenu`); use `mousedown` capture + `preventDefault` on controls so ProseMirror `CellSelection` survives until `deleteRow` / `deleteColumn` run. Table styling is in `styles.css` (e.g. `--nota-table-grid`); CSS for header corners must target `thead` / `tbody` explicitly because TipTap’s table DOM puts `<colgroup>` before row groups.
- TipTap emoji: use `@tiptap/extension-emoji` on the **v2** line with the rest of TipTap (`^2.27.2`); default `enableEmoticons: false`; `suggestion: { allow: () => false }` turns off the `:` suggestion UI while keeping shortcode/Unicode behaviour. Style `span[data-type='emoji']` under `.tiptap-editor` if fallback images misalign. Vitest loads `is-emoji-supported` via the package—stub `HTMLCanvasElement.prototype.getContext` in `vitest.setup.ts` to silence jsdom; colocate `tiptap-emoji.spec.ts` with the editor.
- Nota Pro / RevenueCat: client uses `@revenuecat/purchases-js` (dynamic import helpers in `app/lib/revenuecat/purchases.ts`); dashboard entitlement **Identifier** is `Nota Pro` (`ENTITLEMENT_NOTA_PRO` in `app/lib/revenuecat/constants.ts`). Server parsing of RevenueCat REST v1 `GET /v1/subscribers/{app_user_id}` in `app/lib/revenuecat/subscriber.server.ts` tries the REST API id `entl81a5d697e6` (`ENTITLEMENT_NOTA_PRO_REST_ID`) first, then `Nota Pro`, with `REVENUECAT_SECRET_API_KEY`. The `/notes` layout loader returns `notaProLocked` when the user lacks entitlement (empty notes); the client loader **does not** merge IndexedDB when `notaProLocked` is true. `NotaProGate` blocks the notes shell; note shortcuts, offline sync, and the signed-in command palette respect unlock state from the layout. Gated routes such as `og-preview` and the note detail loader use the same server check. After a successful purchase, invalidate the server subscriber cache (`intent=invalidateNotaProCache` on the layout `action`) and `revalidate()` so loaders refresh.
- Shortcut-related app preferences use the `user_preferences` table (RLS, migration under `supabase/migrations/`); the notes layout loader returns them and the layout `action` branches on `intent=updateUserPreferences`. Zustand `nota-preferences` mirrors the server with `preferencesPendingSync` when toggled offline; `dailyNoteIdByLocalDate` (local calendar `YYYY-MM-DD` → note id) stays device-local only. `openTodaysNoteClient` in `app/lib/open-todays-note.ts` is shared by `use-todays-note-shortcut` and the command palette. Notes history: Mod+[ / Mod+] call React Router `navigate(-1)` / `navigate(1)` from `use-notes-history-shortcut.ts` (capture-phase `keydown` on `document` so shortcuts work inside TipTap; skip when focus is inside `data-nota-command-palette`, same as other global note shortcuts).
- Electron translucency: `BrowserWindow` uses `transparent: true` and `backgroundColor: '#00000000'`; on macOS add `vibrancy: 'under-window'` and `visualEffectState: 'followWindow'` so the window is not a solid black composite. `root.tsx` runs an inline head script that adds `nota-electron` on `<html>` when `window.nota` exists. Use unlayered CSS (not only `@layer base`) for `html.nota-electron` / `body` transparency and for `html.nota-electron .nota-notes-root` so Tailwind utility backgrounds on the notes shell do not override transparency.
